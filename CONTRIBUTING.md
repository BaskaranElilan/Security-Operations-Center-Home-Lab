# Contributing to SOC Home Lab

Thank you for your interest in contributing to this project! This is an educational platform for learning cybersecurity and SOC operations.

## Ways to Contribute

### 🐛 Report Bugs
- Search existing [Issues](https://github.com/BaskaranElilan/Security-Operations-Center-Home-Lab/issues) first
- Provide clear, reproducible steps
- Include your environment details:
  - OS (Windows/Linux/macOS)
  - Azure region
  - Tool versions
  - Exact error messages

### 💡 Suggest Improvements
- Open an Issue with label `enhancement`
- Describe the improvement and why it's useful
- Provide examples if possible

### 📚 Improve Documentation
- Fix typos or unclear sections
- Add missing information
- Improve diagrams or explanations
- Add examples or tutorials

### 💻 Submit Code
- Fork the repository
- Create a feature branch: `git checkout -b feature/your-feature`
- Make your changes
- Add comments explaining complex logic
- Test thoroughly before submitting
- Create a Pull Request with clear description

### 🧪 Test New Features
- Follow installation guide
- Try deployment in your own Azure account
- Document any issues you find
- Share your experience

## Development Setup

```bash
# Clone your fork
git clone https://github.com/YOUR_USERNAME/Security-Operations-Center-Home-Lab.git
cd Security-Operations-Center-Home-Lab

# Create feature branch
git checkout -b feature/your-feature-name

# Make changes
# Test locally
# Commit with clear messages
git commit -m "Add: Clear description of change"

# Push to your fork
git push origin feature/your-feature-name

# Create Pull Request on GitHub
```

## Code Style

- **Shell Scripts**: Use `#!/bin/bash`, follow Google Shell Style Guide
- **YAML**: 2-space indentation, valid YAML syntax
- **Documentation**: Clear headings, code examples, step-by-step instructions
- **Commits**: Descriptive messages, reference issues if applicable

## Pull Request Process

1. Update documentation if needed
2. Add comments to complex logic
3. Test your changes
4. Submit PR with:
   - Clear title and description
   - Reference any related Issues
   - Screenshots if applicable
5. Address feedback from reviewers
6. Wait for approval and merge

## Code of Conduct

- Be respectful and constructive
- Assume good intentions
- Provide helpful feedback
- Welcome diverse perspectives
- Report inappropriate behavior

## Questions?

- Check existing Issues and Discussions
- Review documentation in `/docs`
- Open a new Issue for clarification
